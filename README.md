# Лабораторная работа №3: Изображения и видео

### Обзор проекта
Основная задача данной работы — извлечение существующей разметки объектов из видео (`output.mp4`), где она уже визуализирована, и использование этих данных для обучения собственной модели детекции.

### Реализованные задачи
1. **Декодирование видео**: Видео `input.mp4` и `output.mp4` разложены на последовательность кадров.
2. **Извлечение разметки**:
   - **Метод OpenCV**: Использовано вычитание кадров для нахождения нарисованных Bounding Box'ов.
   - **Метод YOLOv8**: Использована предобученная модель YOLOv8 как эквивалент публичного CV API для сравнения.
3. **Оценка качества**: Проведено сравнение извлеченной разметки с Ground Truth (`annotations.xml`) по метрике IoU.
4. **Форматирование**: Данные сохранены в стандартном формате **COCO**.
5. **Обучение модели**: На основе извлеченных данных обучен детектор **Faster R-CNN** из библиотеки `torchvision`.
6. **Визуализация**: Создано итоговое видео с отрисованной разметкой и сетка предсказаний модели.

### Сравнение методов извлечения (Avg IoU)
| Метод | Средний IoU |
| :--- | :--- |
| **OpenCV (Вычитание)** | 0.4084 |
| **YOLOv8 (CV API)** | 0.5603 |

### Структура файлов
- `src/lab3/extract_frames.py` — Скрипт для извлечения кадров из видео.
- `src/lab3/extract_markup.py` — Скрипт для извлечения боксов и генерации COCO JSON.
- `src/lab3/train_detector.py` — Скрипт для обучения Faster R-CNN.
- `src/lab3/finalize.py` — Скрипт для генерации итогового видео и графиков.
- `src/lab3/extracted_coco.json` — Извлеченная разметка в формате COCO.
- `runs/lab3/extracted_result.mp4` — Итоговое видео с результатами.

### Как запустить
Для воспроизведения результатов необходимо выполнить скрипты в следующем порядке:
1. `python src/lab3/extract_frames.py`
2. `python src/lab3/extract_markup.py`
3. `python src/lab3/train_detector.py`
4. `python src/lab3/finalize.py`

### Выводы и результаты
1. **Эффективность извлечения**: Метод вычитания кадров (OpenCV) позволяет получить разметку без использования нейросетей, однако он чувствителен к шумам и сжатию видео. Использование YOLOv8 как "эталонного" API показало более высокую точность (IoU 0.56 против 0.41), так как глубокое обучение лучше справляется с вариативностью объектов.
2. **Обучение на производных данных**: Лабораторная работа подтвердила гипотезу о возможности обучения моделей на основе визуализированных результатов других систем. Даже при среднем IoU около 0.4, модель Faster R-CNN начинает "схватывать" основные признаки объектов.
3. **Контроль качества**: Метрика IoU является критически важной на этапе извлечения данных, так как ошибки в "добытой" разметке напрямую влияют на финальную точность обученного детектора.
